# 데이터 업데이트 방법

앱은 데이터가 HTML에 내장된 **스냅샷** 구조입니다. 편집 종류에 따라 방법이 다릅니다.

---

## ① 앱 안에서 바로 (소규모 보정) — 재빌드 불필요

지도에서 매장을 클릭해 팝업에서 바로 수정합니다. **브라우저(localStorage)에만 저장**됩니다.

| 편집 | 위치 |
|---|---|
| 직영 ↔ 대리점 | 삼성/베스트샵 매장 팝업의 「직영으로/대리점으로」 |
| 담당팀 배정 | 삼성 매장 팝업의 팀 셀렉트 (그 시군구 전체가 함께 이동) |
| 평수·직원수·매출 | 팝업의 「데이터 입력」 |

⚠️ **반드시 백업**: 다른 PC·브라우저에서는 안 보입니다. 내보내기 탭에서 CSV로 저장하세요.
⚠️ 영구 반영하려면 아래 ④로 CSV를 stores.json 에 합쳐 재빌드해야 합니다.

---

## ② 신규 출점 · 폐점 (매장 증감) — 재빌드 필요

1. `data/stores.json` 을 메모장/VS Code로 열기
2. `"stores"` 배열에서 매장 객체를 추가/삭제 (아래 양식 참고)
3. **build.bat 더블클릭** → 상권분석지도.html 갱신

```json
{
  "id":"SS-9001", "ch":"삼성직영", "nm":"삼성스토어 ○○",
  "ty":"가전종합", "lt":"로드샵",
  "sd":"경기도", "sg":"수원시 팔달구", "em":"인계동",
  "ad":"경기도 수원시 팔달구 ○○로 00",
  "la":37.000000, "ln":127.000000,
  "tm":"경원팀", "mt":"", "c3":0, "nd":0
}
```

`ch` 값: 삼성직영 · 삼성대리점 · 베스트샵직영 · 베스트샵대리점 · 롯데하이마트 · 일렉트로마트

---

## ③ 입주예정 단지 (반기 갱신) — 재빌드 필요

한국부동산원 입주예정물량은 **반기마다** 갱신됩니다 (다음: 2026-08-27).

1. 공공데이터포털에서 최신 CSV 다운로드
   https://www.data.go.kr/data/15111714/fileData.do
2. 단지별 좌표가 필요하면 주소를 지오코딩 (Photon/카카오)
3. `data/apts.json` 의 `"apts"` 배열 교체
4. **build.bat 더블클릭**

```json
{
  "nm":"○○아파트", "ym":"2028-03", "hh":1200, "ty":"분양",
  "ad":"...", "sd":"...", "sg":"...",
  "la":37.0, "ln":127.0, "pr":"동단위"
}
```

앱 실행 중이라면 재빌드 없이 **입주예정 탭 → CSV 업로드**로 임시 추가도 가능합니다(브라우저에만 저장).

---

## ④ 앱에서 수정한 값을 영구 반영하기

①에서 CSV로 내보낸 보정값을 stores.json 에 합치려면:
- 채널 보정 → `직영대리점_분류.csv` 의 운영구분을 stores.json 각 매장의 `ch` 에 반영
- 팀 배정 → `지역별_팀배정.csv` 를 stores.json 의 `tm` 및 `meta.regionTeam` 에 반영

수작업이 번거로우면 CSV를 저에게 주시면 병합해 드립니다.

---

## build.bat 이 안 될 때

- **파이썬 필요**: 이 PC엔 이미 설치돼 있습니다(Python 3.13). 없으면 python.org 에서 설치.
- 실행하면 `backup/` 폴더에 이전 HTML이 자동 보관되므로, 잘못돼도 되돌릴 수 있습니다.
- JSON을 잘못 고치면 빌드가 멈추고 오류 위치를 알려줍니다(깨진 파일이 만들어지지 않음).

---

## 완전 자동 업데이트를 원한다면

매장·입주 데이터를 **주기적으로 자동 수집**하려면 단일 HTML 구조로는 부족하고, 수집 스크립트 + 웹서버 + 스케줄러가 필요합니다. 규모가 커지면 별도로 구성해 드릴 수 있습니다.
