# Google AI Studio로 동일 앱 만들기

## ⚠️ 먼저 알아야 할 제약

AI Studio(Gemini)는 **입력**으로 500KB 데이터를 받을 수는 있지만(1M 컨텍스트), **출력**으로 그걸 다시 뱉을 수는 없습니다(출력 한계 약 6만 토큰). 즉 **데이터가 내장된 완성 HTML을 한 번에 받아내는 건 불가능**합니다.

해결책은 **데이터와 코드를 분리**하는 것입니다.

| 방식 | 내용 | 권장 |
|---|---|---|
| **A. 파일 로더형** | 앱이 실행되면 `stores.json`·`apts.json`을 **드래그&드롭 또는 파일 선택**으로 읽음 | ✅ **권장** |
| B. fetch형 | `fetch('./data/stores.json')` 사용 | 로컬 파일(file://)에서 CORS 차단됨. 웹서버 필요 |
| C. 직접 삽입 | AI가 코드만 만들고, 사용자가 `__DATA__` 자리에 JSON을 직접 붙여넣기 | 가능하나 수작업 필요 |

아래 프롬프트는 **A 방식**(파일 로더 + 직접 삽입 겸용)으로 작성했습니다.

## 1. 준비할 데이터 파일

이 폴더의 `data/` 안에 이미 있습니다. AI Studio에 **첨부**하세요.

| 파일 | 크기 | 내용 |
|---|---|---|
| `stores.json` | 542KB | 매장 1,314개 + 지역-팀 매핑 232곳 |
| `apts.json` | 144KB | 입주예정 단지 675개 |

### stores.json 구조

```json
{
  "meta": {
    "plotted": 1314,
    "asof": "2026-07-14",
    "regionTeam": {
      "대구광역시|수성구": { "tm": "경북팀", "src": "SS기준", "ss": 2, "n": 8 }
    }
  },
  "stores": [
    {
      "id": "SS-0002",
      "ch": "삼성직영",
      "nm": "삼성스토어 강릉",
      "ty": "가전종합",
      "lt": "로드샵",
      "sd": "강원특별자치도",
      "sg": "강릉시",
      "em": "송정동",
      "ad": "강원특별자치도 강릉시 경강로 2380",
      "la": 37.770686,
      "ln": 128.920777,
      "tm": "경원팀",
      "mt": "",
      "c3": 4,
      "nd": 56
    }
  ]
}
```

**필드 설명**

| 키 | 의미 | 값 |
|---|---|---|
| `id` | 매장ID | SS-/LG-/HM-/EM- + 4자리 |
| `ch` | 채널 | 삼성직영 · 삼성대리점 · 베스트샵직영 · 베스트샵대리점 · 롯데하이마트 · 일렉트로마트 |
| `ty` | 매장유형 | 가전종합 · 모바일전문 · 사업장內 |
| `lt` | 입점유형 | 로드샵 · 백화점 · 대형마트숍인숍 · 전자관 · 아울렛 |
| `sd`/`sg`/`em` | 시도 / 시군구 / 읍면동 | |
| `la`/`ln` | 위도 / 경도 | |
| `tm` | 지역팀 | 서울팀·경인팀·경원팀·충청팀·호남팀·경북팀·경남팀 |
| `mt` | 대형팀 여부 | `"대형팀"` 또는 `""` |
| `c3`/`nd` | 3km내 경쟁매장 수 / 최근접 거리(m) | 참고용(앱이 직접 재계산해도 됨) |

### apts.json 구조

```json
{
  "meta": { "total": 675, "src": "한국부동산원 입주예정물량(2025-12-31 기준)" },
  "apts": [
    {
      "nm": "강릉 오션시티 아이파크",
      "ym": "2026-08",
      "hh": 794,
      "ty": "분양",
      "ad": "강원특별자치도 강릉시 견소동 244-2",
      "sd": "강원특별자치도",
      "sg": "강릉시",
      "la": 37.77614,
      "ln": 128.93031,
      "pr": "동단위"

```

`ym`=입주예정월, `hh`=세대수, `ty`=분양/임대/조합, `pr`=좌표 정밀도(법정동 중심 기준)

## 2. AI Studio 프롬프트 (전문 — 그대로 복사해서 사용)

> **모델 권장**: Gemini 2.5 Pro
> **설정**: Temperature 0.2~0.4, Output length 최대
> **첨부**: `stores.json`, `apts.json`

---

### [프롬프트 시작]

당신은 프론트엔드 개발자입니다. 첨부한 두 JSON 파일의 **구조만 참고**해서, **단일 HTML 파일**로 동작하는 가전 유통 상권분석 지도 앱을 만들어 주세요.

**절대 규칙**
1. JSON 데이터를 코드에 그대로 복사해 넣지 마세요. 출력 길이 초과로 실패합니다.
2. 대신 앱 실행 시 **파일을 읽어들이는 구조**로 만드세요. 화면 중앙에 드롭존을 띄우고, 사용자가 `stores.json`과 `apts.json`을 드래그&드롭하거나 파일 선택으로 넣으면 지도가 렌더링됩니다.
3. 동시에, HTML 안에 `<script id="DATA" type="application/json">__DATA__</script>` 와 `<script id="APT" type="application/json">__APT__</script>` 자리표시자를 두어, 사용자가 나중에 JSON을 직접 치환할 수 있게 하세요. 자리표시자에 실제 JSON이 있으면 드롭존을 건너뛰고 바로 렌더링합니다.
4. 외부 라이브러리는 CDN에서 로드: Leaflet 1.9.4, Leaflet.markercluster 1.5.3, leaflet.heat 0.2.0, Chart.js 4.4.1.
5. **localStorage 외의 저장소는 쓰지 마세요.** 서버·백엔드 없음.
6. UI 문구는 전부 한국어. 다크 테마 기본.

**A. 데이터 도메인 (반드시 이대로 구현)**

채널 6종 — 각각 브랜드 색상과 이니셜 배지로 마커 표시:

| 채널 | 이니셜 | 배경색 | 글자색 |
|---|---|---|---|
| 삼성직영 | SS | `#1428A0` | 흰색 |
| 삼성대리점 | SDP | `#00AEEF` | 흰색 |
| 베스트샵직영 | BS | `#A50034` | 흰색 |
| 베스트샵대리점 | BSD | `#EC6C8C` | 흰색 |
| 롯데하이마트 | HM | `#ED1C24` | 흰색 |
| 일렉트로마트 | EM | `#FFD400` | **검정** (가독성) |

마커는 원형 점이 아니라 **가로 26~32px의 둥근 사각 배지**로, 안에 이니셜 텍스트를 넣습니다.

팀 구조 — 두 종류가 **동시에** 존재합니다:
- **지역팀 7개** (서울·경인·경원·충청·호남·경북·경남): 매장의 `tm` 필드. **모든 매장**이 하나의 지역팀에 속합니다.
- **대형팀**: 매장의 `mt === "대형팀"` 인 63개(일렉트로마트에 입점한 삼성 매장). 이들은 **지역팀에도 동시에 속하므로 중복 집계**입니다. 일렉트로마트 채널 자체(73개)는 경쟁사이며 대형팀이 아닙니다.

**B. 화면 구성**

좌측 사이드바(340px) + 중앙 지도 + 우측 슬라이드 패널(330px).

사이드바는 4개 탭:

**탭1 — 필터·검색**
- 채널 6개 토글 칩(브랜드 색상, 각 매장수 표시)
- 담당팀 셀렉트: `<optgroup>`으로 「지역팀」 7개와 「채널 전담」(대형팀) 분리
- 시도 셀렉트 → 시군구 셀렉트 (**팀→시도→시군구 3단 연동**: 상위를 고르면 하위 목록이 그 범위로 자동 축소)
- 입점유형 · 매장유형 다중 선택 칩
- 매장명·주소 검색창 (디바운스 220ms)
- 검색 결과 리스트 (상위 60개, 클릭 시 지도 이동 + 팝업)

**탭2 — 점유율**
- **현재 범위 브레드크럼**: `전국 전체 › 경북팀 › 대구광역시 › 수성구` — 각 단계에 ✕ 버튼으로 그 단계만 해제. 「전국 전체」 클릭 시 일괄 초기화.
- 분석 단위 3버튼: 팀 / 시도 / 시군구
- 기준 채널 셀렉트: 6채널 + 「삼성 전체(직영+대리점)」 + 「베스트샵 전체」
- **점유율 분모 옵션 (토글 2개)**
  - `일렉트로마트 제외` (**기본 켜짐**) — EM 컬럼을 표에서 빼고 분모에서 제외
  - `가전종합 매장만 비교` (기본 꺼짐) — `ty === "가전종합"` 만 분모
- **기준 채널 공백 지역 (3버튼)**: 전체 / 공백지역만 / 공백 제외
  - **시군구 단위**로 판정: 기준 채널 매장이 0개인 시군구 집합을 만들고, 「공백지역만」은 그 지역만, 「공백 제외」는 그 지역을 뺍니다. 지도·검색 리스트에도 함께 적용.
- 누적 막대 차트(Chart.js) + 지역별 매장수·점유율 표
  - 표의 행 클릭 시 **팀 → 시도 → 시군구 단계별 드릴다운**
  - 기준 채널 점유율이 **전체 평균 대비 5%p 이상 낮은 행은 붉게 강조**(공략 후보)
  - 표 하단에 「지역 합계」 행, 그 아래 점선으로 구분된 **「대형팀」 중복 집계 행** (지역 합계에 더하지 말라고 표기)

**탭3 — 입주예정**
- 연도(2026/2027) · 사업유형 · 최소 세대수 필터
- 정렬: 기회점수 / 세대수 / 자사매장 최원거리 / 입주 임박순
- **기회점수 = 세대수 ÷ 최근접 자사매장 거리(km)** — 큰 단지가 자사매장에 가까울수록 높음(판촉 효율). 반대로 **공백 상권**을 찾으려면 「최원거리 순」 정렬.
- 단지 마커: 연도별 색상(2026 `#00E5C7` / 2027 `#C77DFF`), **반지름 = √세대수 / 4** 의 원형 마커, 임대는 점선 테두리
- 단지 클릭 시: 세대수·입주월·최근접 자사매장·거리·3km내 4사 매장수·기회점수 팝업, 「담당매장 보기」 버튼으로 매장 팝업 연결
- CSV 업로드로 단지 추가(2028년 등), 업로드 템플릿 다운로드

**탭4 — 내보내기**
- 지역별 팀 배정 CSV 내보내기 / 업로드 / 초기화
- 직영·대리점 분류 CSV 내보내기 / 업로드 / 초기화
- 매장 평수·직원수·월매출 입력 데이터 CSV
- 현재 필터 결과 전체 CSV
- 지역별 점유율 CSV

**C. 지도 기능**

우측 상단에 접이식 **「지도 옵션」 패널**:
- **배경지도 5종**: 다크(CARTO dark) / 라이트(CARTO light) / 표준(OSM) / 위성(Esri World Imagery) / 지형(OpenTopoMap)
- 표시 토글: 마커 클러스터링 / 매장 밀도 히트맵 / 매장명 라벨 / 경쟁 연결선
- **클러스터 해제 줌 슬라이더** (10~19, 기본 17)
- 범례(6채널 + 입주 연도)
- 빠른 이동: 전국 / 수도권 / 영남 / 호남 / 충청

**클러스터링 — 매우 중요**: 하나의 클러스터 그룹이 아니라 **채널별로 6개의 독립 클러스터 그룹**을 만드세요. 삼성직영은 삼성직영끼리만 묶이고, LG와는 절대 합쳐지지 않습니다. 클러스터 배지도 해당 채널의 브랜드 색상 원형에 `개수 + 이니셜`(예: `12 / SS`)로 표시합니다. `maxClusterRadius: 26`, `disableClusteringAtZoom: 17`.

**경쟁 반경 분석**: 매장 팝업의 「반경분석」 버튼 → 우측 패널에 1/3/5km 원 표시(선택 반경만 채움), 반경 내 타 매장을 **Haversine 거리** 계산해 거리순 리스트, 채널 구성비 표, 자사/타사 매장 수. 반경 내 매장까지 채널 색상의 얇은 연결선.

**D. 편집 기능 (localStorage 저장)**

1. **직영/대리점 보정**: 삼성·베스트샵 매장 팝업에 「직영으로」/「대리점으로」 버튼. 클릭 즉시 마커 색상과 모든 집계 재계산.
   - **예외 규칙(강제)**: 매장명에 `일렉트로마트`가 포함된 삼성 매장은 **항상 삼성직영**입니다. 대리점으로 바꾸려 하면 차단하고 "직영 고정" 표시.
2. **지역 담당팀 배정**: **삼성 매장 팝업에서만** 팀 셀렉트를 노출합니다. 팀을 바꾸면 **그 시군구의 모든 채널 매장이 함께** 이동합니다(`meta.regionTeam`의 `시도|시군구` 키 기준). 타 채널 팝업에는 팀이 읽기 전용으로 표시되고 "팀 변경은 삼성 매장에서만 가능합니다" 안내.
3. **매장 실적 입력**: 평수·직원수·월평균매출·기준기간·비고. 평당매출·인당매출 자동 계산.

수정된 항목은 팝업에 `수정됨` 뱃지, 내보내기 탭에 수정 건수 카운트, 초기화 버튼 제공.

---

**E. 계산 규칙**

- 거리: Haversine 공식, 지구 반경 6,371,000m, 결과는 미터 정수
- 점유율 = (기준 채널 매장수 ÷ 분모 매장수) × 100. 분모는 위의 분모 옵션과 모든 사이드바 필터(채널·팀·시도·시군구·입점유형·매장유형·검색어)를 **전부** 반영합니다.
- 열세 판정: 해당 지역 점유율 < (전체 평균 − 5%p)
